Explicación del problema y solución: Se trata de 2 partes importantes este proyecto, satisfacer una necesidad básica
de tener una base de datos de Clientes, con datos importantes pero escasos y la cual pueda mantenerse, y por otra parte un
diseño de un modelo técnico que intenta evaluar conocimientos al aplicarlos en el proyecto.
La solución se centra en cumplir ambas partes lo que puede estar generando código innecesario o bien un modelo que quizas 
podría mejorarse.

Supuestos :
1- El cliente es único, cuyo rut es la llave para aplicar este supuesto.
2.- Cada vez que se hace una carga de datos, se asume que es una nueva base de datos, por ende, se borra lo anterior.
3.- El RUT puede ser utilizado con o sin puntos decimales, pero si con guión y digito verificador válido.
4.- En clase ArchivoServcio, metodo exportar, no es utilizado en la lógica del proyecto, por ende se generó código 
repetido para cumplir con o requerido.
5.- Se agregarón al menu algunas opciones y funciones para que tenga mayor facilidad para el usuario en la operación.
6.- La opción de Exportar se programo sólo para ambiente Windows.
